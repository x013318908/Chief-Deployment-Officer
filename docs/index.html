Chief-Deployment-Officer

A single-file PHP MCP endpoint that lets your approved AI agent operate files, back up changes, and help ship vibe-coded apps to a remote server.

No server-side install. Upload one PHP file, approve one AI agent, and operate through MCP. CC0.

Download cdo.php View on GitHub

Why single-file?

Many shared hosting environments do not provide SSH, Docker, or modern deployment pipelines. Chief-Deployment-Officer is designed to be uploaded like a normal PHP file, then used by an approved AI agent through MCP.

What it does

Remote file operations

Approved agents can list, read, write, create, rename, delete, copy, stat, and hash files under the endpoint directory.

Human approval

The browser approval flow issues a bearer token only after the user explicitly approves the AI agent.

Deployment support

Runtime diagnostics and production.env placement helpers support shared-host deployment without exposing secret values through MCP.

Quick start

Download cdo.php, rename it to a hard-to-guess PHP filename, upload that single file to a HTTPS server, then open the deployed URL in a browser. The page explains the MCP endpoint, authorization state, and AI-agent connection flow.
